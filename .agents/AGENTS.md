# Code Refactoring Opportunities

After a feature, bugfix, or other modification has been implemented:
1. Analyze the modified code and surrounding codebase to identify refactoring opportunities to improve code quality, reduce duplication, and improve performance.
2. Include a summary of these refactoring opportunities in the `walkthrough.md` artifact.
3. Create a separate, detailed document in the artifact directory (e.g., `refactoring_opportunities.md`) detailing the recommended refactoring steps, code snippets, and rationale, so that the user can use it later for performing the refactoring.
